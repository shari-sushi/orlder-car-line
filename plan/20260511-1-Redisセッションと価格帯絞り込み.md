# Redisセッションと価格帯絞り込み

## 目的・背景

現状は「3行まとめて送る」方式のため会話ステートが不要。
価格帯をスクレイピングして提示し、絞り込みを1ターン追加することで
ステートマシン・Redis・スクレイピングを実際に使う構成にする。

## 会話フロー

```txt
[セッションなし]
  ユーザー: アルファード\n2020\n5万km以下
  ↓ Bot がカーセンサーを取得して価格帯をスクレイピング
  Bot:
    アルファード・2020年・5万km以下の検索はこちら
    https://...

    価格帯は 180万円〜380万円 になりました。
    価格帯で絞り込めます。

    返信例）100万円〜200万円

[step: waiting_price] ← Redis に保存（TTL: 30分）

  ユーザー: 200万円〜300万円
  ↓ Redis からセッション取得 → 価格帯付き URL を組み立て
  Bot: 絞り込んだ URL を返す

[セッション削除 → 初期状態へ]
```

## セッション構造

Redis キー: `session:{user_id}`、TTL: 30分

```json
{
  "step": "waiting_price",
  "data": {
    "car_name": "アルファード",
    "year": "2020",
    "distance_min": "0",
    "distance_max": "50000"
  }
}
```
