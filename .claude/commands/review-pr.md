---
description: GitHub PRをレビューし、コメントを投稿する
---

GitHub Pull Request をレビューし、結果を PR に直接コメントとして投稿してください。

## 引数のパース

`$ARGUMENTS` の先頭トークンが PR の指定、残りがレビュー依頼文です。

- PR番号（例: `123`）→ カレントリポジトリの該当PR
- URL（例: `https://github.com/owner/repo/pull/123`）→ そのリポジトリの該当PR
- レビュー依頼文（例: `セキュリティ観点でレビューして`）→ 省略可能

## 手順

### 1. PR情報の取得

`gh pr view <PR番号またはURL> --json number,title,body,headRefName,baseRefName,author,url` で PR のメタ情報を取得する。

### 2. 差分の取得

`gh pr diff <PR番号またはURL>` で差分を取得する。

### 3. レビュー観点の決定

以下の優先順位でレビュー観点を決定する：

1. **コマンド依頼文**（最優先）: `$ARGUMENTS` から抽出した依頼文があればそれに従う
2. **PR description のレビュー依頼**: PR本文に「レビュー観点」「確認してほしい点」などの記載があればそれを補助的に参照する
3. **デフォルト観点**（依頼文がない場合）: 以下をすべて適用する
   - `.claude/rules/coding-conventions.md` のプロジェクト規約への準拠
   - バグ・ロジックエラーの検出
   - セキュリティ上の問題（OWASP Top 10 等）
   - 可読性・保守性

> コマンド依頼文と PR description の内容が競合する場合は、コマンド依頼文を優先する。

### 4. レビューの実施

差分を精査し、以下の形式で所見をまとめる：

```
## レビュー依頼

### 観点
（今回適用した観点を明記）

---

## レビュー結果

### 総評
（全体的な評価を1〜3文で）

### 指摘事項

#### 🔴 必須修正
- `ファイルパス:行番号` — 問題の説明と修正案

#### 🟡 改善提案
- `ファイルパス:行番号` — 提案内容

#### 🟢 良い点
- （良かった点があれば）
```

指摘事項がない場合は「指摘事項なし」と記載する。

### 5. GitHubへの投稿

以下のコマンドでレビューコメントを投稿する：

```bash
gh pr review <PR番号またはURL> --comment --body "<レビュー内容>"
```

投稿後、PR の URL をユーザーに伝える。
